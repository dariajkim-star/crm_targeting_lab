# SPEC — crm-targeting-lab (P2)

## Why

채용공고 주요업무 1번("고객군을 나누고 타겟팅 전략을 함께 고민")과 우대사항(CRM·마케팅 분석: LTV·이탈예측·캠페인 분석, XGBoost, BI 시각화)을 직격하는 포트폴리오. 이탈 모델 하나로는 부족하다 — **"누구를(세그먼트) 왜(가치×이탈위험) 얼마의 비용으로(캠페인 ROI)"**까지 답하는 CRM 분석 풀사이클이 목표다. 이탈확률만 높은 고객이 아니라 **"잃으면 아까운(가치 높은) × 잃을 위험이 큰"** 고객을 우선하는 2×2 매트릭스가 프로젝트의 중심 논리이며, 산출물은 모델이 아니라 **타겟팅 의사결정 프레임**이다.

## Capabilities

- **CAP-1 RFM 세그멘테이션**
  - **intent:** RFM 지표 + K-means로 고객군을 분리하고(k 선정 근거: elbow/실루엣), 세그먼트별 인구통계·행동 프로파일과 페르소나를 정의한다.
  - **success:** 세그먼트 프로필 리포트와 페르소나 4~6개가 산출된다.

- **CAP-2 이탈위험 분류 모델**
  - **intent:** XGBoost로 이탈위험 확률을 추정한다. 불균형 처리(scale_pos_weight)·교차검증·PR-AUC 병행 평가를 포함한다.
  - **success:** baseline 로지스틱 대비 PR-AUC +15% 이상 리프트를 제시한다.

- **CAP-3 이탈요인 해석**
  - **intent:** SHAP 전역·개별 해석을 세그먼트별 리텐션 액션으로 번역한다(예: 거래 감소형 → 사용처 쿠폰, 한도 불만형 → 한도 상향 제안).
  - **success:** 세그먼트별 이탈요인 top5 + 액션 매핑 문서가 산출된다.

- **CAP-4 LTV 확률 모델 (방법론 데모)**
  - **intent:** Online Retail II 거래 로그에 BG/NBD + Gamma-Gamma(**pymc-marketing**)를 적용해 12개월 기대 LTV를 추정한다. **CAP-5의 2×2와 결합하지 않는 독립 산출물**이다(모집단이 다름).
  - **success:** 고객별 LTV 테이블과 모델 적합도 요약이 산출되고, 2×2와 별개 산출물임이 문서에 명시된다.

- **CAP-5 이탈위험×고객가치 2×2 타겟팅 매트릭스**
  - **intent:** BankChurners 단일 데이터셋 안에서 (이탈위험 = CAP-2 예측확률) × (고객가치 = `Total_Trans_Amt` **단일** 실측 프록시)로 Save 우선 / 관망 / 저비용 유지 / 이탈 수용의 4분면을 분류한다. **합성 LTV 주입 없음** — 두 축 모두 동일 모집단 실측. **`Total_Revolving_Bal`·`Credit_Limit`은 가치 축에 산입하지 않는다** — 세그먼트 프로파일 서술(CAP-1)에만 쓰는 참고 지표다. **(2026-07-21 3차 개정 — 금지 범위 명확화, 1-7 외부 리뷰 반영)**: 이 금지는 **고객가치 산식·LTV·가치 축·가치 기반 우선순위 계산**에 한정된다. 이탈위험 모델(CAP-2)의 **예측자로 쓰는 것은 이 금지의 대상이 아니다** — 단, 예측자로 쓰더라도 가치 지표로 재해석하거나 리텐션 액션의 인과적 근거로 쓰는 것은 금지된다. 이 구분은 코드 주석이 아니라 SPEC이 소유한다. (실제 적용: 1-7이 `Avg_Utilization_Ratio`를 예측자로 채택했고, `Credit_Limit`은 ablation 결과 기여 미미(PR-AUC +0.005·전역 요인 9/9위)로 **제외**했다.) (2026-07-20 개정: 원안의 "보조 지표 활용도" 문구가 AD-11 "고객가치는 `customer_value()` 한 함수만이 정의한다"와 충돌했다. 복합 가중치를 허용하면 2×2의 분면과 캠페인 타겟 순위가 서로 다른 가치 정의를 쓰는 문을 열게 되므로 단일 정의로 확정한다. 보조 지표를 가치 축에 넣으려면 AD-11 규칙대로 `customer_value()`·스키마 문서·CAP-5 한계 문구를 함께 고쳐야 한다.)
  - **success:** 매트릭스 시각화 + 분면별 고객 수·정책 제안이 산출되고, 가치 프록시의 선정 근거와 한계가 명시된다.
  - **(2026-07-20 2차 개정 — 이름 소유권)** 가치 축의 **원천 컬럼명 자체**도 `crm/segment/value.py`가 소유한다. `crm/` 아래 다른 모듈은 문자열·어트리뷰트·타입/스키마 선언·`eval`/`query` 표현식 등 어떤 형태로도 `Total_Trans_Amt`를 직접 명명하지 않으며, 컬럼명 심볼을 `value.py`에서 import하는 것도 금지다. 근거: 1-2 외부 코드리뷰에서 **공개 상수 import가 실제로 AD-11 가드를 우회**함이 실증됐다(위반 0건 통과). "재계산 금지"만으로는 정적 강제가 불가능하고 — `df[X]`의 X가 가치 축인지 판별할 수 없다 — 이름을 봉인해야 규칙이 기계적으로 검사된다. **대가**: 스키마 검증·dtype 맵 등 정당한 메타데이터도 막히며 이는 알고 감수한 비용이다. 실소비자가 실제로 막히는 사례가 나오면 그 근거로 재검토한다. 상세는 AD-11.

- **CAP-6 캠페인 시뮬레이터**
  - **intent:** 기대 절감액 = P(이탈) × 고객가치 × 리텐션 성공률 − 캠페인 비용으로, 고정 예산 제약 하 타겟 우선순위를 산출한다.
  - **success:** 고정 예산 시나리오에서 "무작위 타겟 대비 기대 절감액 X배"를 제시한다. 예산·비용은 **통화 중립**(캠페인 건수 또는 데이터 원 통화)으로 표현한다.

- **CAP-7 민감도 분석**
  - **intent:** 리텐션 성공률·건당 비용 파라미터 그리드에서 ROI 등고선을 산출한다 — 실측 불가한 가정의 강건성을 오프라인으로 평가한다("실험 없이 어디까지 말할 수 있는가").
  - **success:** 민감도 등고선 차트 + "어떤 가정 구간에서 결론이 뒤집히는가" 해석이 산출된다.

- **CAP-8 BI 데이터마트 + Tableau 대시보드**
  - **intent:** 세그먼트 현황 / 이탈 위험 / 타겟팅 3탭을 위한 단일 데이터마트(CSV + 스키마 문서)와 대시보드 사양서를 산출하고, Tableau Public에 공개한다.
  - **success:** 데이터마트 2종·스키마·사양서가 산출된다(세션 산출물). Tableau Public 링크가 README에 연결된다(사용자 실행 — OQ-5). 탭은 고객분석 3탭(`mart_customers`) + LTV 방법론 데모 1탭(`mart_ltv_demo`) = 4탭이다(마트 2분할의 귀결).

## Constraints

- **데이터 분리**: 이탈(BankChurners, 10,127건)과 LTV(Online Retail II, 거래 ~100만 건)는 **레코드 결합 불가** — 결합·합성 매핑 시도 금지. 2×2는 BankChurners 내부 실측만 사용하고, BG/NBD는 별도 데모로 분리한다.
- **라벨 성격 명시**: `Attrition_Flag`는 사후 스냅샷 단면 라벨이다. 산출물 전반에서 "이탈 **위험 분류**(cross-sectional)"로 표기하고, 관측창/예측창 시계열 구조를 흉내내지 않는다.
- **통화 규율**(P1 3-4 교훈): 금액은 **데이터 원 통화 그대로**, 무단 환산 금지. BankChurners는 통화 무단위, Online Retail II는 GBP. 예산 시나리오는 통화 중립 표현.
- **가정 파라미터 명시**: 리텐션 성공률·건당 비용은 실측 불가한 가정이다. 대표값(성공률 30% 등)은 "가정"으로 라벨링하고 정답으로 주장하지 않으며, 강건성은 CAP-7이 담당한다.
- **재현성**: 시드 고정, `features → train → simulate` 파이프라인 스크립트화.
- **데이터마트 계약**: BI는 파이프라인 중간산출물이 아닌 **출처별 마트 CSV + 스키마 문서**만 소비한다(P1 AD-9 정신). 마트는 출처별 2개로 분리한다 — 단일 파일은 위 격리 제약을 최종 산출물에서 무효화하기 때문이다(스파인 AD-2).
- **공개 노출**: Tableau Public은 비공개 저장 경로가 없다 — 퍼블리시하면 마트 내용이 공개 조회 가능해진다. 마트에는 공개 데이터셋 유래 값만 담는다(스파인 AD-10).
- **테스트**: RFM 계산·기대수익 산식 pytest 단위 테스트 필수.
- **LTV 라이브러리**: `lifetimes`는 discontinued(마지막 릴리스 2020-07, 2026-07 확인) — 공식 후속 **pymc-marketing 버전 고정** 사용, 적합 실패 시 코호트 기반 단순 LTV 폴백.

## Non-goals

- 실제 A/B 테스트 (CAP-7 오프라인 민감도 분석으로 대체)
- 실시간 스코어링·서빙 API (P1과 달리 배치 분석 + BI가 소비층)
- 개인화 추천(협업 필터링)
- 두 데이터셋의 레코드 수준 결합, 또는 합성 LTV를 2×2에 주입하는 우회 (구조적으로 불가 + 결론을 오염시킴)
- 시계열 이탈 예측(관측창·예측창 설계) — 라벨 구조상 불가

## Success signal

① 이탈위험 모델 PR-AUC가 baseline 로지스틱 대비 **+15% 이상** 리프트. ② 고정 예산 시나리오에서 **"무작위 타겟 대비 기대 절감액 X배"**를 수치로 제시. ③ Tableau Public 링크 + GitHub README로 **5분 안에 전체 스토리 파악** 가능. ④ 모든 가정(데이터 결합 불가, 라벨 단면성, 성공률·비용 가정, 통화)이 산출물에 명시돼 있다.

## Open questions

- **OQ-5 (외부 의존, 사용자 확인)**: Tableau Public 계정·데스크톱 설치 확보 여부. P1 SAS 선례대로 **분업 설계** — 세션은 데이터마트(단일 CSV + 스키마)와 대시보드 사양서까지 산출하고, 워크북 제작·퍼블리시는 사용자가 수행한다. 미확보여도 데이터마트 산출은 블로킹되지 않는다.

## Assumptions

- Kaggle BankChurners·UCI Online Retail II는 공개 데이터로 다운로드 가능하다.
- `Total_Trans_Amt`가 카드 고객의 가치 프록시로 타당하다(연간 거래액 = 수수료 수익의 1차 동인). 한계는 CAP-5 산출물에 명시한다.
- 리텐션 성공률 대표값 30%는 업계 통용 보수값이며 실측이 아니다(CAP-7이 강건성 담당).
- 스택은 전부 표준 라이브러리(stack.md) — 신규 인프라 불요.
