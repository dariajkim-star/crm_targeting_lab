# Conventions — P1 계승 작업 관례

P1(credit_scorecard-lab)에서 확립·검증된 관례를 P2가 계승한다. 다운스트림 스킬(architecture, epics, dev-story, code-review)은 이 문서를 P1 선례의 압축본으로 읽는다.

1. **SPEC-first 경로**: SPEC(CAP/제약/non-goal) → 아키텍처 스파인(AD-N) → 에픽·스토리 → sprint-status. PRD 없음.
2. **정직성 톤**: 성능 미달·네거티브 결과·가정을 숨기지 않고 산출물에 명시. 네거티브 결과도 산출물로 설계한다(P1 3-2 선례).
3. **실데이터 실행 DoD**: 합성 mock 초록으로 done 금지 — 매 스토리 실데이터 실행이 결론·수치를 만든다(P1 전 에픽에서 실버그 다수 검출).
4. **문서 수치 규율**: 모든 리포트 수치는 재현 가능한 커밋된 코드 경로 + 출처 명시(P1 3-2·3-4 리뷰 교훈). **통화는 데이터 원 통화 그대로**(P1 3-4 통화 오기 사고).
5. **deferred-work.md**: 의도적으로 미룬 항목을 근거와 함께 기록.
6. **3-레이어 코드리뷰**: Blind/Edge/Auditor 병렬 → triage(patch/defer/dismiss) → 스토리 파일 기록.
7. **외부 의존 스토리 분업**: 세션이 실행 못 하는 외부 도구(P1 SAS, P2 Tableau)는 "자동생성·검증은 세션, 실행·퍼블리시는 사용자"로 분업 설계(P1 3-3 선례).
8. **git**: 스토리 단위 커밋, 데이터·아티팩트 gitignore + 재생성 스크립트(NFR5 상당).
9. **옵시디언 미러**: 에픽 단위 요약을 `ob_storage/신용평가_CRM_사이드프로젝트/`에 기록.
