## 1. 판정 규칙 (AD-12)

| 항목 | 값 | 출처 |
|---|---|---|
| 임계 방식 | 분위수(quantile) | `crm/config.py::QUADRANT_RULE` |
| 위험 축 분위수 | **0.75** | config (선험 규약) |
| 가치 축 분위수 | **0.50** | config (선험 규약) |
| 경계 포함 | `>=` 상단 (`upper_inclusive`) | config |
| **실현된 위험 컷** | **0.126842** | **런타임 산출** (config 아님) |
| **실현된 가치 컷** | **3899.0** | **런타임 산출** (config 아님) |

**config에는 분위수 레벨만, 실현된 컷은 런타임 산출.** AD-12는 임계 "방식·값"을 config가 소유하라고
하고 AD-1은 데이터 유래 값을 config에 두지 말라고 한다. 두 규칙을 동시에 지키는 형태는 1-3이 RFM에서
이미 확립했다 — config는 `RFM_QUANTILES=5`(개수)만 갖고 경계는 런타임 산출 후 리포트 기재. 여기도
같다. 실현된 컷은 4-1 마트의 `threshold_official_*` 컬럼으로 나간다(AD-3).

### 왜 위험 축만 0.75인가 (AC5 — 비대칭의 근거)

**가치 축은 중앙값**이다. 2×2의 통상 정의이고, `customer_value` 분포가 완만해(중앙값 3,899 / 평균
4,404) 중앙값이 제 역할을 한다.

**위험 축은 중앙값을 쓸 수 없다.** 실측:

```
churn_prob   median 0.00511   mean 0.19757   |   실제 이탈률 0.1607
             < 0.001 인 행: 2,854 / 10,127
```

중앙값 컷을 쓰면 **이탈확률 0.5% 고객에게 "고위험" 라벨**이 붙는다. 대시보드에서 가장 먼저 지적당할
지점이고, "Save 우선" 분면이 1,273명으로 부풀어 캠페인 예산 논의가 무의미해진다. 75분위는 상단 셀의
의미를 유지한다.

**후보 비교** (가치 축은 중앙값 고정):

| 위험 축 방식 | 실현 컷 | Save 우선 | 채택 여부 |
|---|---|---|---|
| q0.50 (중앙값) | 0.00511 | 1,273 | ✗ 라벨 의미 붕괴 |
| **q0.75** | **0.12684** | **446** | **✓ 채택** |
| q0.80 | 0.37538 | ~307 | 더 보수적이나 세그먼트별 분석 여지 축소 |
| 실제 이탈률 0.1607 고정 | 0.16070 | 415 | ✗ **AD-1 위반** — 0.1607은 라벨에서 읽은 값 |
| q(1−기저율) | 0.77607 | 307 | ✗ 같은 이유로 AD-1 위반 |

0.75는 `SEGMENT_K=4`와 같은 성격이다 — **분석가가 분포를 보고 고른 관례값**이지 적합된 통계량이 아니다.
적합된 값은 실현 컷 0.126842이고, 그것은 config에 들어가지 않는다.

### 이 규칙은 순위만 읽는다 (AC6 — A2와의 결합 여부)

**두 컷 모두 분위수이므로 판정은 `churn_prob`의 순서만 사용하고 보정된 크기는 사용하지 않는다.**

이것이 중요한 이유: 현재 `churn_prob`는 **in-sample·미보정**이다(평균 0.1976 vs 실제 0.1607, 8분위는
약 20배 과대 — `deferred-work.md`의 「calibration 실측」절). 보정 여부는 에픽1 회고 액션 **A2로 아직
미결**이다. 그러나 **강한 단조 증가 변환은 어떤 고객도 분위수를 가로질러 옮기지 못하므로, A2가 어떤
결론으로 닫히든 이 스토리의 판정은 한 건도 바뀌지 않는다.**

`tests/campaign/test_matrix.py::test_a_strictly_increasing_transform_of_risk_changes_nothing`이 이
성질을 기계적으로 고정한다. 이 테스트가 깨지면 규칙이 크기를 읽기 시작한 것이고 A2가 3-1로 들어온
것이다.

> ⚠️ **3-2는 사정이 다르다.** 기대절감액 = `P(이탈) × 고객가치 × 성공률 − 비용`은 확률을 **곱셈의
> 크기**로 쓴다. A2는 3-2 착수 전에 닫아야 한다. 이 리포트의 면제는 3-1에만 적용된다.

## 5. 가치 프록시의 근거와 한계 (NFR1 재명시)

가치 축은 `crm/segment/value.py::customer_value()` 단일 정의를 **소비**한다(AD-11 — 재계산·재가중 없음).

**[가정]** 연간 거래액이 카드 고객의 가치 프록시로 타당하다 — 수수료 수익의 1차 동인이고, BankChurners가
직접 측정한 유일한 지출량이다.

**한계** (1-2 리포트에서 이월, 여전히 유효):

1. **거래량은 수익성이 아니다.** 마진·자금조달비용·연체손실이 반영되지 않는다.
2. **보조 지표 미반영** — `Total_Revolving_Bal`·`Credit_Limit`은 SPEC CAP-5에 따라 가치 축에 산입하지
   않는다(세그먼트 프로파일 서술 전용). 3차 개정(2026-07-21)이 이 금지의 범위를 "가치 산식·LTV·가치 축·
   가치 기반 우선순위"로 한정했고, **이탈 모델의 예측자 사용은 별개 사안**임을 명시했다.
3. **단면 스냅샷** — 미래 가치가 아니라 과거 12개월 실적이다. BG/NBD 기대 LTV는 에픽2의 **독립** 산출물이며
   모집단이 달라 이 2×2와 결합하지 않는다(AD-1).
4. **분면 경계는 이 모집단 상대값** — 다른 고객군에 그대로 옮길 수 없다.

## 6. 이 스토리가 만들지 않은 것

- **파이프라인 단계 없음** — `matrix.py`는 순수 함수다. 공식 수치의 자리는 4-1 마트 + 4-3 Tableau 탭이고,
  이 문서는 **세션 리포트**다(1-2 M3 판정의 연장, sprint-status 헤더 M2 주석).
- **`target_priority` 없음**(3-3) · **기대절감액·예산·비용 없음**(3-2) — `matrix.py`는 예산 개념을
  구조적으로 알지 못하며 AD-9 구조 가드가 이를 강제한다(`AD-9 campaign order`, 실스캔 1건 / 위반 0).
- **마트 컬럼 배선 없음**(4-1) — 단 `quadrant_thresholds()`가 실현 컷을 반환하므로 4-1이
  `threshold_official_*`을 채울 수 있다.
