## 2. 잘 된 것

### 2-1. 회고→스토리 연쇄가 실제로 돌았다 (에픽1 A2 → 3-0) — 이 에픽의 최대 자산

### 2-2. 실측이 인계 전제를 반증했다 (3-3 → 3-4, D3)

3-3 create-story는 *"3-4가 `RETENTION_GRID` 하단을 쓸면 절감액이 뭉개져 동점 발생 → 동점 규칙을 실데이터로 검증"*이라 넘겼으나, 3-4가 **그리드 전역 25점에서 동점 0건**을 측정. `saving = P·value·rate − cost`는 `rate>0`에서 `P·value`의 엄격 단조 **아핀 변환**이고 아핀은 distinctness를 보존하므로, `P·value`가 10,127 distinct인 한 어떤 `(rate,cost)`에서도 동점이 없다. **리포트는 "동점을 실검증했다"고 쓰지 않았다 — 그렇게 쓰면 거짓이니까.** 에픽1의 "문서가 실측을 앞서지 않기" 규율이 에픽3에서도 작동했다.

### 2-3. 가드가 못 잡는 것을 정직하게 인정했다 (함정 4)

3-3 사전조사가 두 parquet **행 오정렬**을 실제로 밟아 합계가 **+37.2%**(1,994,741 vs 1,454,088)로 부풀었는데 **어떤 예외도 안 났다** — 둘 다 평범한 `RangeIndex`라 `bc.index.equals(sc.index)`가 True. 3-2가 확률 오선택(+19.0%)을 막으려 Series 이름 검사까지 넣었는데, 더 큰 오차가 아무 가드에도 안 걸린 것.

**John (PM):** "그리고 코드리뷰 Auditor가 create-story의 '4-1에서만 실효' 진술까지 **반증**했습니다. `target_priority`는 절감액 축이 정체성을 안 들고 다녀서 4-1 마트 형태에서도 무예외 통과합니다. 스스로 적은 완화책을 스스로 깎아냈어요 — 포트폴리오 서명이 이런 겁니다."

### 2-4. risk_quantile annex의 물리 분리 (3-4, D2 b′)

분면 정의 민감도를 FR14 성공률×비용 등고선과 **물리 분리된 별도 산출**로 넣고, **official 컬럼·마트 write 0건을 테스트로 단언**했다. 실측: 분면 정의가 rate·cost보다 결론을 더 흔든다(0.70→0.80에서 save_first 537→348, −35%). 프레임은 "판정을 무르는 게 아니라 견고함 검산" — 이것이 본공연 워싱 스크리너 배제 임계 민감도의 리허설이다.
